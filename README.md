# cc-company-kr

Claude Code에서 가상 회사 조직을 구축・운영하는 플러그인 **(한국어판)**

`/company`를 실행하면 비서가 창구가 되어, 당신의 사업에 최적화된 조직(부서)을 구축합니다. 일상 운영에서는 비서에게 말을 거는 것만으로 CEO가 적절한 부서에 업무를 배분합니다.

---

## 설치

```
/plugin marketplace add <your-github-username>/cc-company-kr
/plugin install company@cc-company-kr
```

---

## 컨셉

```
당신（오너）
    │
  CEO（배분・의사결정）
    │
  ┌─┴──────────────────────┐
  │         │              │
비서실    개발팀         리서치팀
（상시）
```

---

## 사용 방법

### 처음 실행

```
/company
```

인터뷰를 통해 사업・목표・필요한 부서를 선택하면 `.company/` 폴더가 자동으로 생성됩니다.

### 이후 운영

다시 `/company`를 실행하면 비서 모드로 시작됩니다.

```
비서: 안녕하세요! 오늘은 무엇을 도와드릴까요?

당신: 오늘 할 일 알려줘
당신: React hooks에 대해 조사해줘
당신: 새 기능 설계 문서 만들어줘
당신: 이번 주 회고하고 싶어
```

---

## 선택 가능한 부서

| 부서 | 역할 |
|------|------|
| 비서실 ※상시 | TODO 관리, 브레인스토밍, 빠른 캡처, 일일 관리 |
| CEO ※상시 | 의사결정, 부서 배분 로직 |
| 리뷰 ※상시 | 주간・월간 회고 |
| PM | 프로젝트 진척, 마일스톤, 티켓 관리 |
| 리서치 | 시장 조사, 경쟁사 분석, 기술 조사 |
| 마케팅 | 콘텐츠 기획, SNS 전략, 캠페인 |
| 개발 | 기술 문서, 설계서, 디버그 로그 |
| 경리 | 청구서, 경비, 매출 관리 |
| 영업 | 클라이언트 관리, 제안서 |
| 크리에이티브 | 디자인 브리프, 브랜드, 에셋 |
| 인사・채용 | 채용 관리, 온보딩, 팀 관리 |

## 생성되는 폴더 구조

```
.company/
├── CLAUDE.md                 # 조직 전체 규칙
├── secretary/                # 비서실
│   ├── inbox/                # 빠른 캡처
│   ├── todos/                # 일일 태스크 (YYYY-MM-DD.md)
│   └── notes/                # 브레인스토밍 메모
├── ceo/
│   └── decisions/            # 의사결정 로그
├── reviews/                  # 주간 리뷰
├── engineering/              # 개발 (선택 시)
│   ├── docs/
│   └── debug-log/
├── research/                 # 리서치 (선택 시)
│   └── topics/
└── ...                       # 기타 선택된 부서
```
